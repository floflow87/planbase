Je souhaite faire évoluer l’architecture IA de PlanBase.

Aujourd’hui, plusieurs fonctionnalités IA existent (OpenAI + Ollama + Whisper), mais elles sont encore fragmentées. Mon objectif est de transformer cela en un système IA cohérent, contextuel et réellement utile dans les workflows produit.

---

# 🎯 OBJECTIF GLOBAL

Passer de :
- fonctionnalités IA isolées

à :
- un assistant intelligent unifié, capable de :
  - comprendre le contexte projet / client / document
  - proposer des actions concrètes
  - aider à produire (tickets, notes, roadmap, décisions)
  - améliorer la rentabilité et la gestion

---

# 🧪 ÉTAPE 1 — AUDIT & CORRECTIONS

Avant toute amélioration, je veux :

1. Vérifier la cohérence entre :
   - OpenAI (résumés, extraction, CRM)
   - Ollama (chat assistant)
   - Whisper (transcription)

2. Identifier les incohérences :
   - logique dupliquée (résumé / recommandations / actions)
   - prompts différents pour des usages similaires
   - données non partagées entre moteurs

3. Vérifier :
   - que toutes les routes IA sont sécurisées (requireAiAccess)
   - que les erreurs sont bien gérées (timeout, modèle absent, etc.)
   - que les réponses sont structurées (pas de texte brut inutilisable)

4. Centraliser les prompts :
   - créer un système de prompt templates réutilisables
   - éviter les prompts écrits en dur dans plusieurs fichiers

---

# 🧱 ÉTAPE 2 — UNIFIER LE MOTEUR IA

Créer une couche centrale :

## AI Orchestrator

Un service unique qui gère :
- quel moteur utiliser (OpenAI vs Ollama)
- quel prompt utiliser
- quel contexte injecter

Exemple :

aiOrchestrator.run({
  type: "project_analysis",
  context: {...},
  provider: "auto"
})

Règles :
- Ollama → chat temps réel / assistant
- OpenAI → tâches complexes / structuration / extraction
- Whisper → input vocal uniquement

---

# 🧠 ÉTAPE 3 — CONTEXTE INTELLIGENT (CRITIQUE)

Aujourd’hui, l’IA fonctionne mais manque de profondeur contextuelle.

Je veux enrichir le contexte automatiquement :

## Contexte injecté selon la page :

### Projet
- budget
- marge
- temps consommé / restant
- statut
- livrables
- tâches

### Client / CRM
- historique des échanges
- statut commercial
- projets liés

### Notes / Documents
- contenu
- tags
- type de document

---

## IMPORTANT

Ne jamais demander à l’utilisateur de fournir le contexte manuellement.
