🚀 Prompt Replit — Système d’automatisations Slack (type Notion)

Je souhaite implémenter un système d’automatisations Slack dans mon application (PlanBase), inspiré de Notion.

Objectif :

- permettre de créer des automatisations Slack :
  - au niveau global (settings)
  - au niveau local (projet, backlog, CRM, roadmap, etc.)
- déclencher des notifications Slack en fonction d’événements métier
- garder une architecture scalable, centralisée et maintenable

---

🧠 1. Vision produit

Je veux un système où :

- chaque entité (projet, backlog, roadmap, pipeline CRM…) peut avoir ses propres automatisations
- il existe aussi des automatisations globales
- les automatisations sont basées sur :
  - un trigger
  - des conditions
  - une action (Slack)

---

🧱 2. Architecture globale

A. Event-driven system (obligatoire)

Créer un système d’événements central :

type AppEvent =
  | "backlog.created"
  | "backlog.updated"
  | "backlog.prioritized"
  | "backlog.completed"
  | "project.created"
  | "project.updated"
  | "roadmap.updated"
  | "crm.deal_created"
  | "crm.deal_won"
  | "crm.stage_changed";

Chaque action métier importante doit déclencher un event via :

emitEvent(event: AppEvent, payload: any)

---

B. Automation Engine

Créer un service central :

handleEvent(event: AppEvent, payload: any)

Ce service :
- récupère les automatisations actives
- filtre selon conditions
- exécute les actions (Slack)

---

🗄️ 3. Modèle de données

Créer une table :

automations

Champs :

- id
- organization_id
- scope_type :
  - global
  - project
  - backlog
  - roadmap
  - crm
- scope_id (nullable si global)
- name
- is_active (boolean)

Trigger :
- event_type (ex: backlog.prioritized)

Conditions (JSON) :

{
  "priority_status": "prioritized",
  "project_id": "xxx"
}

Action :
- action_type = "slack_message"

Slack config :
- slack_webhook_url (ou référencer settings)
- channel (optionnel)
- message_template (string dynamique)

Metadata :
- created_at
- updated_at
- created_by

---

⚙️ 4. Niveaux d’automatisation

A. Global (settings)

Accessible via :
Settings → Automations

- scope_type = global
- s’applique à toute l’organisation

---

B. Local (contextuel)

Accessible depuis :
- page projet
- page backlog
- page roadmap
- page CRM

Bouton :
“⚡ Automatiser”

Comportement :
- scope_type = project / backlog / etc.
- scope_id = id de l’entité

🔁 7. Exécution

Lors d’un event :

1. récupérer toutes les automations :
   - globales
   - + celles du scope (ex: projet_id)

2. filtrer :
   - event_type match
   - conditions match

3. envoyer message Slack

🧠 9. Règles importantes

- ne pas dupliquer les automations
- centraliser la logique dans Automation Engine
- ne jamais appeler Slack directement depuis les modules métier
- tout passe par emitEvent
