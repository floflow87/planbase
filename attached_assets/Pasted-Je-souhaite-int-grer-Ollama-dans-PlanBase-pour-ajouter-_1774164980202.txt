Je souhaite intégrer Ollama dans PlanBase pour ajouter une première couche d’intelligence artificielle utile, simple et non intrusive.

IMPORTANT :
- ne pas créer une usine à gaz
- intégrer l’IA de manière contextuelle dans les bons endroits
- privilégier des use cases concrets et actionnables
- garder une UX fluide, rapide et compréhensible
- ne pas casser l’existant
- implémenter une V1 robuste et évolutive

## Objectif V1

Créer un assistant IA basé sur Ollama capable de :

1. répondre aux questions générales sur l’application (assistant produit)
2. analyser un projet et donner des recommandations business (rentabilité, organisation, priorités)
3. résumer des notes et documents
4. proposer des améliorations concrètes sur des contenus existants

## Stack attendue

- Backend Node.js existant
- Ollama en local (via API HTTP)
- modèle recommandé :
  - llama3 ou mistral (léger pour V1)
- architecture modulaire :
  - ai.service.ts
  - ai.routes.ts
  - ai.controller.ts

  - contexte utilisateur
  - contexte projet

- model: "llama3"

# 2. ASSISTANT GLOBAL (CHATBOT)

Créer un module assistant accessible depuis :

- bouton flottant
- ou sidebar

Fonctionnalités :

### 1. Questions produit
Exemples :
- "comment créer un projet ?"
- "comment améliorer mon organisation ?"
- "comment utiliser le CRM ?"

### 2. Questions business projet
Si un projet est sélectionné :

- injecter contexte :
  - nom projet
  - budget
  - temps consommé
  - marge
  - statut

Exemples de questions :
- "comment améliorer la rentabilité ?"
- "quels sont les risques sur ce projet ?"
- "que devrais-je faire en priorité ?"

### 3. Conseils proactifs

Le bot doit pouvoir suggérer :

- actions prioritaires
- points d’attention
- bonnes pratiques

# 3. PROMPT SYSTEM (IMPORTANT)

Rôle :
Tu es un assistant produit et business intégré à PlanBase.

Objectifs :
- aider l’utilisateur à mieux gérer ses projets
- améliorer sa rentabilité
- structurer son travail
- proposer des actions concrètes

Style :
- clair
- actionnable
- orienté produit / business
- éviter le blabla

Toujours inclure :
- recommandations concrètes
- points d’attention
- actions suivantes
